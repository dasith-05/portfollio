# Reflective Report: Group 21 Portfolio Project

## Project Overview
This project involved the collaborative design and development of a professional group portfolio website. The primary goal was to create a centralized platform to showcase 10 group members, featuring their names, academic streams, and short biographies. The development focused on using core web technologies—**HTML5, CSS3, and JavaScript**—to ensure a lightweight, responsive, and interactive user experience.

## Development Process

### 1. Structural Foundation (HTML)
The project began with establishing a robust HTML structure. We focused on:
* **Professional Layout:** Creating a clean container system for the team profiles.
* **Data Placeholders:** Setting up specific fields for Names, Academic Streams, Bios, and Profile Image placeholders.
* **Scalability:** Expanding the structure to accommodate 10 distinct member profiles while maintaining code readability.

### 2. Styling and Interactivity (CSS)
Visual appeal and user engagement were prioritized through custom CSS:
* **Interactive Cards:** Implemented hover effects on profile cards to provide visual feedback and a modern feel.
* **Alignment & Layout:** Utilized Flexbox/Grid to center profile pictures and ensure uniform alignment across all 10 cards.
* **Navigation Elements:** Added a dedicated "Prompts" button next to the team section to redirect users to the project documentation.

### 3. Dynamic Functionality (JavaScript)
JavaScript was utilized to transform a static page into a dynamic application:
* **Data Management:** Instead of hardcoding every detail in HTML, member names and biographies were injected via JavaScript for better maintainability.
* **Slider/Carousel Logic:** Developed a custom slideshow mechanism that supports both manual scrolling and automatic transitions.
* **Smooth Navigation:** Enhanced the internal linking and scrolling behavior to ensure a fluid transitions between sections.

## Reflective Summary
The development of this portfolio allowed the group to practice modular coding. By separating the data (JS) from the presentation (CSS) and structure (HTML), we created a site that is easy to update. The implementation of the manual/automatic slider was the most challenging aspect, requiring precise timing logic in JavaScript to ensure it didn't interfere with user manual scrolls.
